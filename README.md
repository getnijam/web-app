# nijam-web

Frontend for **Nijam**, a Vite + React SPA behind auth. Talks to [`nijam-api`](../api) via a generated, fully-typed client.

License: BSL 1.1. See [`../plans/CLAUDE_WEB_PLAN.md`](../plans/CLAUDE_WEB_PLAN.md) for full conventions.

## Setup

The API client in `src/client/` is **generated** from the backend's OpenAPI spec, it does not exist until you generate it. Run the API first, then:

```bash
npm install
cp .env.example .env          # VITE_API_URL + NIJAM_SPEC_URL
npm run gen:api               # generate src/client/ from the running API's /openapi.json
npm run dev                   # http://localhost:5173
```

`gen:api` is **manual**, run it whenever the API contract changes (it's not wired into `dev`). The API serves its spec at the live `/openapi.json` endpoint (there's no spec file on disk), so the API must be running and `NIJAM_SPEC_URL` must point at it (e.g. `http://localhost:8787/openapi.json`).

## Conventions

- **Never hand-write `fetch()`**, call the generated SDK functions (`login`, `signup`, …) and the TanStack Query `*Options`/`*QueryKey` helpers from `src/client/`.
- **Errors**: the API returns `{ error: { code, message, field? } }`. Render `error.message` directly; map `error.field` to the offending form input.
- **No `localStorage`/`sessionStorage`**, use React state, TanStack Query cache, or URL state. The session lives in the `nijam_session` cookie, set by the API; the frontend never reads it.
- **Every query renders four states**: loading, error, empty, data (`src/components/states/`).
- Styling is Tailwind + shadcn-style primitives only.

## Auth flows implemented

- `/signup` → creates account, shows "check your inbox"
- `/login` → signs in, invalidates `/me`, routes to `/onboarding`
- `/verify?token=…` → verifies email; handles success, used (treated as success), expired (resend), invalid
- `/_authed/*` → `beforeLoad` gates on `/v1/auth/me`, redirects to `/login` on 401
- Sign out → `logout` mutation, invalidates `/me`, back to `/login`

## Layout

```
src/
  routes/                 # TanStack Router file-based routes
    __root.tsx, index.tsx, login.tsx, signup.tsx, verify.tsx
    _authed/route.tsx, _authed/onboarding.tsx
  components/ui/          # shadcn-style primitives (button, input, label, card)
  components/states/      # LoadingState, ErrorState/ErrorBanner, EmptyState
  hooks/useMe.ts
  lib/                    # utils (cn), api-error helpers
  client/                 # GENERATED by Hey API, do not edit by hand
  main.tsx
```
