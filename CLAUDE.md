# Nijam Web, Claude instructions

Frontend for **Nijam**, a Vite SPA that talks to the Nijam API. The public surface (landing `/`, legal `/terms` `/privacy`, and the auth pages) lives here alongside the auth-gated dashboard (`_authed/*`). No SSR; SEO for public pages is done **client-side** via TanStack Router's route `head` + `<HeadContent />` (see the SEO convention below). Optimize for clarity, type safety, a fast dev loop.

> `../plans/` is temporary (removed after v0.1). This file is the durable source of truth for the web repo.

## Stack (locked, ask before changing)

- **Build**: Vite · **Node**: 22.
- **Framework**: React **19** (function components; `ref` is a normal prop, shadcn components rely on this, so we are NOT on React 18).
- **Language**: TypeScript, strict.
- **Routing**: TanStack Router (file-based; `routeTree.gen.ts` is generated on dev/build).
- **Server state**: TanStack Query, via Hey API's generated `*Options`/`*Mutation`/`*QueryKey` helpers (not literal hooks).
- **API client**: Hey API generated (`@hey-api/openapi-ts` + `@hey-api/client-fetch`) into `src/client/` (committed; never hand-edit).
- **UI**: shadcn/ui (`radix-luma` style, `hugeicons`) on the unified `radix-ui` package.
- **Styling**: **Tailwind v4**, CSS-first (`@tailwindcss/vite`, no `tailwind.config.*`/PostCSS). Tokens live in `src/styles/globals.css` under `@theme inline`. Font: **DM Sans** for all UI text (`--font-sans` + `--font-heading`); **Geist Mono** for code only (`--font-mono`).
- **Forms**: react-hook-form + `@hookform/resolvers/zod`.
- **License**: BSL 1.1.

## Layout

```
src/
  routes/                 # file-based; _authed/ is the protected layout (beforeLoad → /v1/auth/me, redirect /login on 401)
  components/
    ui/                   # Flex, Grid, Text (our primitives) + shadcn button/card/input/label
    states/               # LoadingState, ErrorState (+ ErrorBanner), EmptyState, use everywhere
    auth/                 # AuthLayout, AuthBrandPanel, Logo
  hooks/  lib/  styles/
  client/                 # GENERATED by Hey API, do not edit by hand
  main.tsx                # client.setConfig({ baseUrl: VITE_API_URL, credentials: 'include' }) + providers
```

## Conventions

- **No bespoke CSS, style with shadcn first, then Tailwind. (Major rule.)** Order of preference for ANY styling/UI: **(1)** reach for a **shadcn** component (`Button`, `Card`, `Badge`, `Separator`, `Tabs`, …) plus our `Flex`/`Grid`/`Text` primitives; **(2)** if none fits, use **Tailwind utility classes** (the scale + theme tokens); **(3)** only if a value genuinely can't be expressed, add a `@theme` token / `@utility` in `globals.css`. **Never hand-author a `.css` file or bespoke class rules for a feature**, if you're writing selectors, you've skipped step 1 or 2. (The home page's `home.css` was deleted and rebuilt on Tailwind/shadcn for exactly this reason.)
- **Layout/text via primitives, not raw tags.** Use `Flex`/`Grid` instead of `<div className="flex/grid …">`, and `Text` instead of `<h1>/<p>/<span>`. `Text` is variant-driven: `variant` (display/h1-h6/lead/body/caption/code/inherit), `color` (default/muted/primary/secondary/success/warning/danger/info), `weight`, `align`, `truncate`, polymorphic `as`. `Grid` takes `cols` as a number or a responsive array mapped to `[base,sm,md,lg,xl,2xl]` (e.g. `cols={[1,1,2,2,3]}`), prefer it over hand-written `grid-cols-*` breakpoint classes. Genuinely non-layout/decorative or conditionally-displayed (`hidden lg:flex`) elements may stay raw.
- **Buttons, always the shadcn `Button`, never anything else.** Every clickable button is the shadcn `Button` (`@/components/ui/button`) with the right `variant`/`size`, **never a raw `<button>`, and never `<Flex as="button">` / `<div role="button">`.** `Button` is already `inline-flex items-center justify-center gap`, so it owns button layout; tune the look via `className` (twMerge lets your classes win over the variant defaults) instead of rebuilding it. It also has a `loading` prop → disables + shows an inline spinner (use `loading={mutation.isPending}` rather than hand-rolling). For a button that navigates, compose `<Button asChild><Link to="…">…</Link></Button>`. (`Flex`/`Grid` are layout-only and must never stand in for an interactive button.)
- **Internal navigation uses TanStack Router `<Link to="…">` (SPA), never a raw `<a href>` to an in-app route** (a full reload defeats the SPA). Reserve `<a>` for external URLs (`target="_blank" rel="noopener noreferrer"`) and in-page `#anchors`. The shadcn `Button` composes with it via `asChild`: `<Button asChild><Link to="…">…</Link></Button>`.
- **Every route sets a `head`.** From `@/lib/seo`: **public** routes use `head: () => seo({ title, description, path, noindex? })` (full title/description/OpenGraph/Twitter/canonical); **dashboard** (`_authed/*`) routes use `head: () => privateSeo('Page')` (title + `noindex`, no OG/canonical, these pages are never indexed or shared). The `_authed` layout (`_authed/route.tsx`) sets a `privateSeo('Dashboard')` default so every dashboard page is titled + `noindex` even if a leaf is missed; leaf routes override the title. The root route (`__root.tsx`) renders `<HeadContent />` (React 19 hoists the tags) and supplies `baseHead()` (a default title + description fallback). Public auth/utility pages (`/signup`, `/verify`, `/forgot-password`, `/reset-password`, `/invite`) pass `noindex: true`. `index.html` carries static homepage fallbacks tagged `data-seo-default` for non-JS scrapers; `main.tsx` strips them on boot so the router owns the head (no duplicate tags). The social image is `public/og.png` (1200×630).
- **Custom hooks: check usehooks.io first.** Before hand-writing any hook, check the **usehooks.io** catalog (<https://www.usehooks.io>). If one fits, **vendor it** into `src/hooks/` via copy-paste (`npx usehooks-cli@latest add <name>`) with a source comment, not a runtime dep (see `src/hooks/use-is-mobile.ts`). Only hand-write a hook when nothing there meets the need, and say so in a comment (e.g. `src/hooks/use-in-view.ts`).
- **Icons come from Hugeicons.** Use `<HugeiconsIcon icon={SomeIcon} size={…} className="text-…" />` with icons from `@hugeicons/core-free-icons`. Check the set first (it's large); only hand-author an inline `<svg>` when Hugeicons genuinely lacks the glyph (e.g. the Nijam brand mark). Never build a one-off icon module when a named Hugeicon exists.
- **Brand & color come from the one preset.** Reuse the existing tokens in `globals.css` (`--primary`, `bg-brand`, the status tokens) as-is; never introduce a new brand hue or a parallel set of tokens for a new surface, scope styles to the surface and reference the preset. Reuse the shared **`Logo`** component (`@/components/auth/Logo`) for the brand mark/wordmark everywhere; don't fork, recreate, or restyle it per page.
- **API calls only through the generated client.** Never hand-write `fetch()`. `gen:api` is **manual** (not wired into `dev`): run `npm run gen:api` with the API running (defaults to `http://localhost:8787/openapi.json`, override via `NIJAM_SPEC_URL`) whenever the API contract changes.
- **Use Hey API's generated TanStack Query helpers, never duplicate them.** For every operation use the generated `xxxMutation()` / `xxxOptions()` / `xxxQueryKey()` from `@/client/@tanstack/react-query.gen`, `useMutation({ ...xxxMutation(), onSuccess, onError })` and `useQuery(xxxOptions(args))`. Never hand-write a `mutationFn`/`queryFn`/`queryKey` that just re-calls a client fn (the generated `mutationFn` already sets `throwOnError: true`, so `onError` fires). `mutate(...)` takes the typed request options (`{ body }`/`{ path }`), not loose args.
- **Errors**: API returns `{ error: { code, message, field? } }`. Render `error.message` directly; map `error.field` to the offending form input via `form.setError`. Helpers in `src/lib/api-error.ts`.
- **Toasts via `notify` (from `@/lib/notify`) for transient feedback.** Use it wherever it makes sense, confirming a succeeded mutation (`notify.success(title, { description })`), surfacing a non-form error (`notify.error`), etc. `notify` wraps sonner with our progress-bar toast (auto-closes ~5s, pauses on hover/focus, has a close button); the `<Toaster />` host is mounted once in `main.tsx`. Don't import sonner's `toast` directly, and don't toast validation/field errors that already render inline under inputs.
- **Every query renders four states**: loading, error, empty, data (`src/components/states/`).
- **Error monitoring (Better Stack, via the Sentry SDK)**: `initSentry()` (`src/lib/sentry.ts`) runs in `main.tsx`, **prod builds only** (dev = true no-op unless `VITE_MONITORING_DEV=true`). Sends to Better Stack via `VITE_SENTRY_DSN` (a public client key, inlined at build; no DSN → no init). Errors + tracing + **replay-on-error** (all text/inputs masked). Top-level `Sentry.ErrorBoundary` + the router `defaultErrorComponent` (both render `FullPageError`); user context set in `_authed/route.tsx`, cleared in `useLogout`. Source maps upload to **Better Stack** via `@sentry/vite-plugin` (pointed at Better Stack with `SENTRY_URL`/`SENTRY_ORG`/`SENTRY_PROJECT`/`SENTRY_AUTH_TOKEN` from Errors → Applications → Advanced settings), runs only when the token is in the build env (Vercel); never commit it.
- **Auth**: read the current user with `useQuery({ ...getMeOptions(), retry: false })`; `_authed/route.tsx` gates via `beforeLoad`; on logout invalidate the `/me` query and navigate to `/login`. The session lives in the `nijam_session` cookie set by the API, the frontend never reads/writes it.
- `tsconfig.json` uses `paths` **without** `baseUrl` (deprecated); the runtime `@/*` alias is resolved by Vite.

## Current scope (v0.1)

Auth flows: `/login`, `/signup`, `/verify`, `/forgot-password`, `/reset-password`. Signed-in users land on **`/projects`** (`_authed/projects.index.tsx`), an **owner-scoped** project list + a create form (name + optional description/repositoryUrl/defaultBranch) that surfaces the new project's UUID + reporter snippet. Each project links to **`/projects/$projectId/runs`** (`_authed/projects.$projectId.runs.tsx`, owner-scoped run list; "No Branch Info" when `branch` is null). The `_authed` layout has a shared header (logo + sign out). No embedded trace viewer yet.

## Guard rails, do NOT

- ❌ **`localStorage` / `sessionStorage` / any browser storage**, required by this environment. Use React state, TanStack Query cache, or URL state. **Single exception (user-approved):** the theme preference may use `localStorage` under the key `nijam-theme` (see `src/components/theme/ThemeProvider.tsx` + the no-flash script in `index.html`). Nothing else may touch browser storage.
- ❌ SSR frameworks (Next/Remix/Astro), Vite SPA only. ❌ React Router/Wouter, TanStack Router only.
- ❌ axios/ky/ofetch, Hey API native fetch. ❌ Hand-write or hand-edit `src/client/`, always regenerate.
- ❌ Redux/Zustand/Jotai/MobX for server state, TanStack Query owns it; try props + URL state before any global store.
- ❌ styled-components/emotion, Tailwind only. ❌ MUI/Chakra/Mantine/Ant, shadcn only. ❌ re-add `tailwind.config.*` (v4 is CSS-first).
- ❌ **Arbitrary `[…]` Tailwind values, never any `[px]`/`[rem]`/`[#hex]`/`[calc()]`/`min-[…]:` custom value.** Use the scale only: spacing via fractional utilities (`size-9.5`, `w-66`, `gap-4.5`, `px-1.75`), fonts via `text-xs/sm/base…`, widths via `max-w-3xl`-style tokens. If a value genuinely isn't in the scale, **add a config token** in `src/styles/globals.css`, a `@theme` var (e.g. `--container-content: 75rem` → `max-w-content`) or an `@utility` (e.g. `bg-brand`/`bg-avatar`/`bg-overlay`), then use that. _Exempt_ (not magic numbers): Tailwind state/variant brackets (`data-[state=open]`, `has-data-[…]`, `content-['']`, `aria-[…]`) and the shadcn preset's own `ui/button|card|input` classes.
- ❌ **Hand-written responsive/grid classes.** Responsiveness comes from the `Flex`/`Grid` primitives, `Grid`'s `cols` array (`cols={[1,1,2,2,3]}`), `gap`, `align`, `justify`, and `Flex`'s `direction`/`wrap`/`align`/`justify`. Never write `grid-cols-*`/`grid-rows-*` breakpoint classes or custom `grid-cols-[…]`/`grid-rows-[…]` templates in `className`; extend the primitive instead.
- ❌ **No ternary hell, one ternary level, ever.** Never nest ternaries (`a ? b : c ? d : e`, or a ternary inside another's branch). Use a **lookup object**, an **early-return helper**, or `if`/`else` instead. Enforced by ESLint `no-nested-ternary`. A single `cond ? a : b` is fine.
- ❌ **No IIFEs or functions defined inside `return`/JSX.** Never write `{(() => { … })()}` in render (or anywhere). Lift render branches and computed values into a **named function** (or a `let` + `if`/`else`) declared **above** the `return`, then use it: `{renderBody()}`. Inline event handlers (`onClick={() => …}`) and `.map((x) => <Row/>)` callbacks are still fine.
- ❌ **Colors used directly**, never `text-white`/`bg-black/60`, raw hex/`oklch()`/`rgb()`, or palette classes (`text-red-500`, `bg-emerald-400`). **Always** use the semantic theme tokens: `bg-background/card/popover/muted/primary/secondary/accent`, `text-foreground/muted-foreground/primary/success/warning/info`, `text-destructive`, `border-border`, `ring-ring`, and `text-primary-foreground` for text on brand-colored surfaces. A new color/palette is defined **once** in `globals.css` (a `:root` var or `@utility`) and referenced by token/class, never inlined in a component.
- ❌ **Hand-rolled UI primitives.** Never write a custom `<input>`, `<kbd>`, `<button>`, `<select>`, dialog, etc. with bespoke styling. **First check the shadcn registry and install it**, `npx shadcn@latest add <name>` (use `yes n | … --yes` to decline overwriting our customized `ui/*` files), then import from `@/components/ui/*`. Compose richer UI from those primitives (e.g. the top-bar search = `InputGroup` + `InputGroupInput` + `Kbd`, not a styled `<input>`/`<kbd>`). The only bespoke primitives we own are the layout/typography ones (`Flex`/`Grid`/`Text`); everything else comes from shadcn.
- ❌ **Hand-built charts.** Never draw a chart yourself (SVG/`<canvas>`/div bars). Charts use **Recharts** via the shadcn `chart` component, `ChartContainer`/`ChartTooltip`/`ChartTooltipContent` from `@/components/ui/chart` (installed with `npx shadcn@latest add chart`), which is what the preset uses. Put series colors in a `ChartConfig` with theme tokens (`color: 'var(--success)'`) and reference `fill="var(--color-<key>)"`; a custom tooltip _content_ component (to show extra fields) is fine, hand-rolling the chart itself is not.
- ❌ date-fns/moment/dayjs for simple cases, try `Intl.DateTimeFormat` first.
- ❌ **Em dashes (U+2014) or en dashes (U+2013) anywhere, never generate one.** Not in user-facing copy (titles, descriptions, button/label text, placeholders, toasts, error and empty-state messages, marketing copy, legal pages) and not in code or comments. Use a comma, colon, parentheses, or split into two sentences for prose; a plain hyphen-minus (`-`) is only for compound words, ranges, and IDs. This is a hard rule: every shipped surface must contain zero em/en dashes.
- ❌ Add deps without confirming · write tests in v0.1 unless critical · change stack/routing/these rules without asking.
